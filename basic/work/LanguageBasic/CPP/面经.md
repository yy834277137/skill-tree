1、一个指针可以是volatile吗：
如果不加 volatile 会发生什么？（编译器的优化陷阱）

编译器在编译 C 代码时，为了追求运行速度，会开启优化。编译器会“自作聪明”地认为：

“这个 bufferPtr 变量，在主循环里被读取了好几次，中间没有代码修改它，那我就不用每次都去内存里读取了，直接把它放到 CPU 的寄存器（临时存储）里复用就行了。”
然而，中断服务程序恰恰是在主程序“看不见”的地方（异步发生）修改了内存中的 bufferPtr。

这就导致了问题：

内存中的 bufferPtr 已经被 ISR 改成了新的值。
但主循环中的 CPU 寄存器里还保存着旧的值。
主循环继续使用寄存器里的旧指针去操作 buffer，导致数据错乱或程序崩溃。

4、拷贝构造函数和赋值运算符的区别
特性	        拷贝构造函数	                                    赋值运算符
调用时机	     创建一个新对象，并用另一个对象初始化它。	            两个对象都已存在，将右边的值赋给左边。
对象状态	     左侧对象尚未存在（正在构造）。	                      左侧对象已经存在（已被构造）。
函数签名	     ClassName(const ClassName& other);	             ClassName& operator=(const ClassName& other);
默认行为	     编译器默认提供（逐个成员拷贝）。	                   编译器默认提供（逐个成员拷贝）。
临时对象	     通常不处理自我赋值。	                             必须处理自我赋值（a = a）。
返回值	         无（构造函数没有返回值）。	                          通常返回 *this 的引用，支持链式赋值。

5、用C++设计一个不能被继承的类
方法1:
class Base final {
    // 类定义
public:
    void func() {}
};

// 错误：无法从 'final' 类 Base 继承
class Derived : public Base {
    // 编译错误
};

int main() {
    Base b;
    b.func();
    return 0;
}

方法2:
在C++98标准中没有 final 关键字，通常利用友元和虚继承来实现。

核心思想：

创建一个辅助模板类（或普通类），将其构造函数设为私有。
将目标类设为辅助类的友元，这样只有目标类能构造辅助类。
让目标类虚继承这个辅助类。
任何试图继承目标类的类，在构造时都必须负责构造辅助类（因为虚继承体系下，最终派生类要负责初始化最底层的虚基类）。但由于辅助类的构造函数是私有的，除了目标类之外，其他类无法访问，从而导致编译错误。

#include <iostream>

// 辅助类：构造函数私有
class MakeFinal {
private:
    MakeFinal() { std::cout << "MakeFinal构造" << std::endl; }
    // 关键：让目标类成为友元
    friend class FinalClass;
};

// 目标类：虚继承 MakeFinal
class FinalClass : virtual public MakeFinal {
public:
    FinalClass() { std::cout << "FinalClass构造" << std::endl; }
    void test() {}
};

// 尝试继承
class Attempt : public FinalClass {
public:
    Attempt() { // 编译错误！
        // Attempt 必须负责构造 MakeFinal
        // 但 MakeFinal 的构造函数是私有的，Attempt 无权访问
    }
};

int main() {
    FinalClass obj; // OK
    obj.test();
    
    // Attempt obj2; // 编译错误
    return 0;
}

方法3:
#include <iostream>

template <typename T>
class FinalHelper {
private:
    FinalHelper() = default; // 构造函数私有
    friend T; // 只有模板参数 T 可以访问
};

// 最终类
class MyClass : public virtual FinalHelper<MyClass> {
public:
    MyClass() { std::cout << "MyClass构造" << std::endl; }
};

// 尝试继承
class TryChild : public MyClass {
public:
    TryChild() {
        // 编译错误：TryChild 需要构造 FinalHelper<MyClass>
        // 但 FinalHelper 的构造函数是私有的
    }
};

int main() {
    MyClass obj; // OK
    return 0;
}
