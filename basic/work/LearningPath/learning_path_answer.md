# 学习路线与面试题：查缺补漏

本文档根据您的技能树，旨在帮助您巩固优势、弥补潜在短板，并为技术面试做好充分准备。

## 一、 学习路线

### 第一阶段：巩固与深化 C++ 与工程化

您已经是 C/C++ 的熟练使用者，此阶段的目标是迈向“专家”级别，并完善您的工程化能力。

1.  **深入 C++ 现代特性 (C++11/14/17)**
    *   **学习重点**:
        *   智能指针 (`std::unique_ptr`, `std::shared_ptr`) 的原理与应用场景。
        *   std::unique_ptr
        原理: 1. 独占所有权, 同一时间只能有一个unique_ptr指向某个对象
             2. 移动语义，不能拷贝，只能通过std::move转移所有权
             3. 性能优势，无额外开销，大小通常和裸指针相同
             template<typename T>
             class unique_ptr {
                 T* ptr;
             public:
                 explicit unique_ptr(T* p = nullptr) : ptr(p) {}
                 ~unique_ptr() { delete ptr; }
                 
                 // 禁用拷贝
                 unique_ptr(const unique_ptr&) = delete;
                 unique_ptr& operator=(const unique_ptr&) = delete;
                 
                 // 支持移动
                 unique_ptr(unique_ptr&& other) noexcept : ptr(other.ptr) {
                     other.ptr = nullptr;
                 }
                 
                 T* operator->() const { return ptr; }
                 T& operator*() const { return *ptr; }
             };
             颜色表
        *   右值引用、移动语义 (Move Semantics) 和完美转发，理解其如何优化性能。
        *   Lambda 表达式的各种用法。
        *   多线程库 (`std::thread`, `std::mutex`, `std::condition_variable`, `std::atomic`)。
    *   **推荐资源**:
        *   书籍: 《Effective Modern C++》
        *   课程: Bilibili/YouTube 上的 C++ 现代特性教程。

2.  **提升软件架构与设计模式**
    *   **学习重点**:
        *   重新审视常用的设计模式（工厂、单例、观察者、策略模式等）在嵌入式场景的应用。
        *   学习面向对象设计原则 (SOLID)。
    *   **推荐资源**:
        *   书籍: 《设计模式：可复用面向对象软件的基础》、《代码整洁之道》

3.  **完善工程化与 CI/CD**
    *   **学习重点**:
        *   **静态/动态代码分析**: 深入使用 `Valgrind`，并了解 `AddressSanitizer (ASan)` / `ThreadSanitizer (TSan)`。
        *   **单元测试**: 学习使用 `Google Test` 框架为您的 C++ 模块编写单元测试。
        *   **CI/CD**: 学习使用 `GitLab CI` 或 `Jenkins` 搭建一个简单的自动化构建、测试流程。

### 第二阶段：强化算法与数据结构

这是面试的重点，也是提升解决问题能力的关键。

1.  **系统性复习**
    *   **学习重点**:
        *   **数据结构**: 数组、链表、栈、队列、哈希表、树（二叉树、平衡树）、图。
        *   **算法**: 排序（快排、归并）、查找（二分）、递归、回溯、动态规划、贪心算法。
    *   **推荐资源**:
        *   网站: [LeetCode](https://leetcode.cn/) (从 "探索" -> "LeetCode 初级算法" 开始)
        *   书籍: 《剑指 Offer》、《代码随想录》

2.  **专项练习**
    *   **练习重点**:
        *   **哈希表**: 解决查找、计数、去重问题。
        *   **二叉树**: 遍历（前/中/后序）、层序遍历、深度/高度计算。
        *   **动态规划**: 从简单的一维 DP 问题开始，如爬楼梯、最大子数组和。

### 第三阶段：补强网络与操作系统

嵌入式系统离不开底层知识。

1.  **操作系统 (OS)**
    *   **学习重点**:
        *   进程与线程的区别与联系。
        *   进程间通信 (IPC) 方式：管道、消息队列、共享内存。
        *   内存管理：虚拟内存、分页、分段。
        *   死锁的产生条件与避免方法。
    *   **推荐资源**:
        *   书籍: 《现代操作系统》
        *   课程: MIT 6.S081 (有 Bilibili 翻译)

2.  **计算机网络**
    *   **学习重点**:
        *   TCP/IP 五层模型，特别是 TCP 和 UDP 的区别。
        *   TCP 的三次握手与四次挥手。
        *   HTTP/HTTPS 的基本原理。
    *   **推荐资源**:
        *   书籍: 《计算机网络：自顶向下方法》

### 第四阶段：提升 Python/Shell 技能

将 Python/Shell 从“工具”提升为“能力”。

1.  **Python**
    *   **学习重点**:
        *   使用 Python 进行文件处理、数据解析 (如解析测试日志)。
        *   学习使用 `pyserial` 与硬件交互。
        *   学习 `pytest` 框架，编写自动化测试脚本。
2.  **Shell**
    *   **学习重点**:
        *   熟练掌握 `grep`, `awk`, `sed` 三剑客。
        *   编写更健壮的 Shell 脚本，包括错误处理、函数定义等。

## 二、 精选面试题

### C++ / 工程化

1.  `std::shared_ptr` 是如何实现引用计数的？它是线程安全的吗？为什么？
2.  解释一下什么是右值引用和移动语义？它解决了什么问题？
3.  在多线程编程中，`std::mutex` 和 `std::atomic` 有什么区别？分别适用于什么场景？
4.  你的项目中使用了 OpenMP 和 SIMD，能具体讲讲你是如何发现性能瓶颈并进行优化的吗？比如，你是如何分析 Cache Miss 的？
5.  如果让你为一个嵌入式模块（比如你做过的“脏图矫正”算法）设计一套单元测试，你会怎么做？会关注哪些测试点？

### 算法与数据结构

1.  给定一个巨大的文件（无法一次性加载到内存），包含大量整数，如何找到出现次数最多的那个数？
2.  实现一个 LRU (Least Recently Used) Cache。
3.  给定一个二叉树和一个目标值，找出所有从根节点到叶子节点的路径，使得路径上所有节点的和等于目标值。
4.  手写一个快排算法，并分析其时间复杂度和空间复杂度。

### 操作系统 / 计算机网络

1.  进程和线程有什么本质区别？为什么说线程是轻量级的进程？
2.  一个程序从编译到最终运行，操作系统在背后都做了哪些工作？
3.  解释一下 TCP 的三次握手过程。为什么需要三次，两次不行吗？
4.  在浏览器地址栏输入一个 URL 后，到页面显示出来，整个过程发生了什么？(DNS 解析 -> TCP 连接 -> HTTP 请求 -> ...)

### 综合/项目

1.  在你过往的项目中，最有挑战性的一个是哪个？你是如何解决其中遇到的技术难题的？
2.  你提到你搭建了测试框架，能详细介绍一下它的架构吗？它解决了什么具体问题？带来了哪些收益？
3.  在进行性能优化时（例如 CPU 全屏成像链路优化），你是如何平衡优化效果、开发成本和代码可维护性的？
4.  你未来的职业规划是什么？你希望在技术上朝哪个方向深入发展？
