三、技术复习清单（按 JD 优先级排序）

A. Camera Pipeline（最高优先级）

  • 能画一张图：Sensor → CSI/MIPI → 驱动 → V4L2 → 用户态 Buffer →（可选 ISP）→ 编码/显示。
  • 说清：RAW / YUV / RGB 常见格式与何时转换；分辨率、stride、时间戳 对同步的意义。
  • 与现经历衔接：安检侧是 X 射线探测器 + 校正/融合，消费侧是 可见光 Sensor + ISP；共同点：帧流水、缓冲、实时性与调试方法论。
  • 对照个人笔记：ISP/DSP/Sensor 链路、basic/work/LearningPath/skills_checklist.md 中第一节。

B. GStreamer

  • pipeline / bin / element / pad / caps negotiation 失败时怎么查。
  • appsink/appsrc 与算法模块对接的典型用法（概念即可）。
  • 与 FFmpeg 对比：何时用管道、何时用 libav*（各一句）。

C. FFmpeg（突击可掌握）

  • ffmpeg 常用：解封装、转码、缩放、截图 示例各记一条。
  • 封装格式（MP4/MKV）与 编码（H.264/H.265）分层表述。
  • 知道 NALU、I/P/B 与延迟关系（checklist 第二节已有，迁移复习）。

D. 编解码与封装

  • I/P/B、GOP、参考帧对 码率与延迟 的影响。
  • 硬编解码与软解的取舍（功耗、延迟、兼容性）。

E. Sensor 与调试（对齐 JD）

  • V4L2：buffer 队列、ioctl 大致流程、mmap/userptr/dmabuf 差异（你学习文档里有）。
  • 常见问题：花屏、帧率不稳、同步异常 的排查顺序（驱动 → 格式 → 时钟 → 带宽）。
  • 若问 ISP：复习 曝光、增益、3A、畸变 等概念级（skills_checklist）。

F. 性能与稳定性（你的差异化优势）

  • 准备 1 分钟版 性能优化案例：perf/cache/OpenMP/SIMD、前后数据。
  • 稳定性：冒烟、自动化回归、内存下降 如何与版本交付挂钩。

G. 业务场景（录像 / 拍照 / 回放）

  • 各用 一条数据流 描述：触发 → 采集 → 处理 → 编码/解码 → 存储或显示。
  • 回放：文件读取 → 解封装 → 解码 → 渲染与时间轴同步（概念）。

四、行为面与项目叙述（建议各准备 1 个 STAR）

从简历中优先选与多媒体最相关的 2～3 个故事，每个控制在 90 秒 内：

  1. 性能优化：背景（全屏链路瓶颈）→ 手段（profiling、并行、SIMD）→ 结果（ms 级、内存）。
  2. 算法量产：脏图矫正或背散射 / 畸变 —— 问题定义 → 方案 → 量产验证。
  3. 多媒体基建：DSP TTK / 冒烟与回归 —— 质量与效率，对应 JD「集成与稳定性」。
  4. 早期 GStreamer + V4L2：摄像头链路 bring-up，体现 Pipeline + Sensor 侧 经验。
